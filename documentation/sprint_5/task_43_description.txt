TODO**Что было сделано и как? (краткое описание)**
CI - Continuous Integration - непрерывная интеграция.
CI - пушим код в репозиторий и у нас запускается валидация, тесты и т.д и наш код,
ничего не поломав, интегрируется с уже существующим кодом.

Можно сделать pipeline - последовательность действий.
Пример: commit -> build -> tests -> code analyse (W : SonarQube) -> CD

Инструменты для CI/CD - Jenkins, GitLab CI/CD , GitHub Actions

TODO**Зачем это было сделано? Чему научился в процессе выполнения задания?**
CI необходим для автоматизации интеграции. Это ускоряет разработку, позволяет разработчикам чаще
делать коммиты и соответственно легче фиксить проблемы, если они возникают
