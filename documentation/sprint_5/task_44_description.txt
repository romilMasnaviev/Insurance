This software design principle states that designs,
solutions, systems, and products work best if they're kept simple.
It means we should favor simplicity over complexity

TODO**Что было сделано и как? (краткое описание)**
KISS - keep it simple, stupid (делай проще, глупец).

Была прочитана статья, а также просмотрено несколько роликов.
Из статьи:
Чем проще - тем лучше.
Делайте методы, классы простыми. Методы, классы должны делать только одну функцию, но делать её
хорошо.
Достоинства:
Делает код проще для изменения и поддержки в будущем.
Делает код понятнее.
Упрощает тестирование этого кода.

Из роликов:
Не реализуйте того, чего не просит бизнес.
Обходитесь минимально возможным количеством кода.
Не используйте огромный фреймворк из-за одного простого метода. Вы утяжеляете код +
добавляете в него лишние зависимости.
Подгоняя решение идеально под заказчика вы делаете его очень хрупким.
