Протокол сети - набор правил, которые должны соблюдать все участники этой сети.
Их поддержка встраивается либо в аппаратную часть (железо), либо в программную часть (код).

Для взаимодействия протоколов между собой существует модель OSI (Open System Interconnection).
Модель основана на разделении системы на 7 отдельных уровней.
7. Прикладной уровень (application layer)
6. Уровень представления (presentation layer)
5. Сеансовый уровень (session layer)
4. Транспортный уровень (transport layer)
3. Сетевой уровень (network layer)
2. Канальный уровень (data link layer)
1. Физический уровень (physical layer)

На 7 уровне информация представляется в виде данных, на 1 в виде бит. Для того чтобы
отправить что-то, информация должна пройти все 7 уровней и превратиться из данных в
биты. Этот процесс называется инкапсуляцией. Компьютер, который получит эти биты должен
превратить их обратно в данные. Этот процесс называется декапсуляцией. На каждом уровне
информация представляется в виде блоков данных протокла - PDU (Protocol data unit).
Пройдемся по каждому из уровней:

1. Физический уровень (physical layer)
Устройство физического уровня оперируют битами. Они передаются по кабелям (оптоволокно) или
без (WiFi, Bluetooth, 4G и т.д)

2. Канальный уровень (data link layer)
Второй уровень решает проблему переадресации при передаче информации.
Он получает биты и превращает их в кадры (frame). Задача - сформировать frame
с адресом отправителя и получателя и отправить их по сети.
У канального уровня есть два подуровня - MAC и LLC. MAC занимается присвоением физических
адресов, LLC занимается проверкой/исправлением данных, а также их отправкой.
На втором уровне OSI работают коммутаторы. Их задача - передать fram'ы от одного MAC адреса на другой.
На канальном уровне используется протокол ARP (Address Resolution Protocol — протокол определения адреса).
С помощью него MAC адреса сопоставляются с IP адресами.

3. Сетевой уровень (network layer)
Задача третьего уровня - маршрутизация. Для этого на третьем уровне работают маршрутизаторы
(роутеры). Роутеры получает MAC адрес с предыдущего уровня и занимаются построением маршрутов.

4. Транспортный уровень (transport layer)
Все 7 уровней модели OSI можно разделить на два группы:
Media layers (уровни среды) - используются для передачи между устройствами (1, 2, 3 уровни)
Host layers (уровни хоста) - используются непосредственно на устройствах
4 уровень выступает посредником между этими двумя группами и относится скорее к Host layer.
Его задачей является транспортировка пакетов. Как раз здесь используются протоколы TCP/UDP
TCP - протокол, чувствительный к потере данных (трехстороннее рукопожатие, пакеты не должны теряться).
При передаче через него данные делятся на сегменты (часть пакета).
UDP - протокол, нечувствительный к потере данных (просто отправляет пакеты адресату).
При передаче через него пакеты уже делятся на датаграммы (самодостаточные куски пакета со всеми заголовками).
Они могут идти разными маршрутами, а часть может теряться.

Первые четыре уровня - специализация сетевых инженеров. Остальными уже занимаются разработчики.

5. Сеансовый уровень (session layer)
Этот уровень отвечает за установление, управление и завершение сеансами связи. Примером может
служить видео звонок по сети.

6. Уровень представления (presentation layer)
Уровень отвечает за кодирование/декодирование данных, преобразование протоколов, а также за
представление видео/фото/аудио.

7. Прикладной уровень (application layer)
Уровень, с которым взаимодействуют пользователи. Задача этого уровня в том, чтобы пользователь
увидел данные в понятном для него виде. Обеспечение взаимодействия пользовательских приложений с сетью.


На данный момент модель OSI устарела, т.к.протоколы работают сразу на нескольких уровнях.
Поэтому на данный момент самой популярной моделью является модель TCP/IP (Transmission Control Protocol/Internet Protocol),
или протокол управления передачей (данных)/интернет-протокол.

В модели TCP/IP располагается уже 4 уровня (хотя они являются объединением уровней из модели OSI).
4. Прикладной уровень (application layer)
3. Транспортный уровень (transport layer)
2. Межсетевой уровень (internet layer)
1. Канальный уровень (link layer)

1. Канальный уровень (link layer)
Назначение этого уровня - дать описание как будет осуществляться передача информации на уровне
сетевых устройств. Самый распространенный протокол на этом уровне - Ethernet

2. Межсетевой уровень (internet layer)
Глобальная сеть состоит из множества локальных. Именно обеспечением связи на этом уровне
занимается данный уровень. Как раз здесь используется протокол IP

3. Транспортный уровень (transport layer)
TCP/UDP
Протоколы транспортного уровня не интерпретируют информацию, полученную с верхнего
или нижних уровней, они служат только как канал передачи

4. Прикладной уровень (application layer)
Здесь для большинства приложений созданы свои протоколы.
Например, браузеры используют HTTP для передачи гипертекста по сети,
почтовые клиенты — SMTP для передачи почты, FTP-клиенты — протокол FTP
для передачи файлов, службы DHCP — протокол назначения IP-адресов DHCP и так далее.

https://selectel.ru/blog/wp-content/uploads/2022/12/tcpip-2048x1056.png

Для обеспечения корректной работы протоколов различных уровней в сетевых моделях
используется специальный метод — инкапсуляция. Суть этого метода заключается в добавлении
заголовка протокола текущего уровня к данным, полученным от протокола вышестоящего уровня.
