TODO**Что было сделано и как? (краткое описание)**
Сделал новые классы:
CoreResponse - ядро ответа (по логике мы обязываем все Response классы иметь
как минимум поля из CoreResponse. В данном случае это лист ошибок)
ValidationError - класс ошибки валидации, имеющий два поля
Validator - класс, инкапсулирующий логику валидации Request

Дальше мы добавляем в Сервис класс Validator. Он будет отвечать за валидацию
приходящих запросов. В нем есть публичный метод validate, который в свою очередь
вызывает методы валидации для полей Request.

Также написал тесты для метода validate (пока что только кейсы, связанные с
валидацией поля FirstName - пустое поле, null поле, нормальные данные)
